## Original Description

Integrity360 Europe found Reflectiz can support PCI DSS v4.0.1 requirements for payment-page script inventory, integrity, and tamper detection.
## Key Points
- Integrity360 Europe은 Reflectiz가 결제 페이지 스크립트 인벤토리, 무결성 및 변조 감지를 위한 PCI DSS v4.0.1 요구 사항을 지원할 수 있음을 확인했습니다.
- 독립적인 PCI 평가자가 새로운 PCI DSS 규칙에 대해 Reflectiz를 테스트했습니다.

## Excerpt
독립적인 PCI 평가자가 새로운 PCI DSS 규칙에 대해 Reflectiz를 테스트했습니다. 평결은 다음과 같습니다. 여기에서 전체 QSA 평가를 확인하세요. → 고객이 결제 시 카드 번호를 입력하면 브라우저가 코드보다 훨씬 더 많이 실행됩니다. 분석 태그, 태그 관리자, 지원 위젯, 결제 iframe: 최신 체크아웃은 수십 개의 제3자 스크립트를 로드하며 그 중 하나를 스키머로 전환할 수 있습니다. 이것이 Magecart가 작동하는 방식입니다. Sansec은 웹 스키밍과 공급망 공격으로 피해를 입은 사이트가 100,000개가 넘는 것으로 집계했습니다. 2018년 영국항공 침해 사고만으로도 38만 건의 거래가 노출되었고 벌금은 1억 8300만 파운드부터 시작되었습니다. 위험한 부분: 악성 코드는 일반적으로 이미 승인한 스크립트를 통해 도착합니다. 공격자가 타사 공급업체를 손상시키고 페이로드는 몇 달 동안 실행한 스크립트에 포함됩니다. 새로운 것은 없습니다. 변경된 것은 페이지에서의 존재가 아니라 스크립트의 동작입니다. PCI DSS v4.0.1은 이제 완전히 시행되는 두 가지 요구 사항으로 이러한 격차를 해소합니다. 6.4.3에서는 모든 결제 페이지 스크립트의 목록을 작성하고 승인하며 무결성을 증명한다고 말합니다. 11.6.1에서는 브라우저가 페이지 콘텐츠와 HTTP 헤더를 수신할 때 해당 내용의 변조를 감지한다고 말합니다. 지속적으로 변경되는 수백 개의 스크립트에 걸쳐 수동으로 수행되므로 확장되지 않습니다. Reflectiz 데이터에 따르면 결제 페이지 스크립트의 약 30%가 2주 이내에 변경됩니다.
