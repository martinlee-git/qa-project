## Summary
Miasma는 4개의 GitHub 조직에서 73개의 Microsoft 리포지토리를 공격하여 액세스를 비활성화하고 오픈 소스 신뢰 위험을 노출시켰습니다. Microsoft의 GitHub 리포지토리는 진행 중인 Miasma 자체 복제 공급망 공격 캠페인의 가장 최근 피해자가 되었습니다. OpenSourceMalware에 따르면 이 사건은 Azure, Azure-Samples, Microsoft 및 MicrosoftDocs를 포함한 GitHub 조직 4곳의 73개 Microsoft 리포지토리에 영향을 미쳤습니다. 개발에는 해당 리포지토리에 대한 액세스를 비활성화하는 GitHub가 있습니다. "Azure/azure-functions-host" 리포지토리에 액세스하려고 하면 "GitHub 서비스 약관 위반으로 인해 GitHub 직원이 이 리포지토리에 대한 액세스를 비활성화했습니다."라는 메시지가 표시됩니다.

## Excerpt
Microsoft의 GitHub 리포지토리는 진행 중인 Miasma 자체 복제 공급망 공격 캠페인의 가장 최근 피해자가 되었습니다. OpenSourceMalware에 따르면 이 사건은 Azure, Azure-Samples, Microsoft 및 MicrosoftDocs를 포함한 GitHub 조직 4곳의 73개 Microsoft 리포지토리에 영향을 미쳤습니다. 개발에는 해당 리포지토리에 대한 액세스를 비활성화하는 GitHub가 있습니다. "Azure/azure-functions-host" 저장소에 액세스하려고 하면 "GitHub의 서비스 약관 위반으로 인해 GitHub 직원이 이 저장소에 대한 액세스를 비활성화했습니다."라는 메시지를 읽습니다. "저장소 소유자인 경우 GitHub 지원에 문의하여 자세한 내용을 확인할 수 있습니다." OpenSourceMalware에 따르면 이번 사건의 영향을 받은 리포지토리 중 일부는 다음과 같습니다. 최신 캠페인에서 주목할만한 점은 지난 달 TeamPCP에 감염되어 Linux 시스템에 정보 유출자를 전달한 "durabletask" PyPI 패키지를 다시 손상시킨 것입니다.
