## Summary
8 Packagist 패키지는 악성 package.json 스크립트를 숨겨 설치 및 작업 흐름 중에 Linux 바이너리 실행을 가능하게 합니다. 새로운 "조정된" 공급망 공격 캠페인은 GitHub 릴리스 URL에서 검색된 Linux 바이너리를 실행하도록 설계된 악성 코드를 포함하여 Packagist의 8개 패키지에 영향을 미쳤습니다. 소켓은 “영향을 받은 패키지는 모두 컴포저 패키지였지만 악성코드는 작곡가.json에 추가되지 않았다”고 말했다. "대신 PHP 코드와 함께 JavaScript 빌드 도구를 제공하는 프로젝트를 대상으로 package.json에 삽입되었습니다." PHP 종속성을 검색하는 개발자와 보안 팀은 Compo에만 집중할 수 있기 때문에 이러한 "생태계 간 배치"는 활동을 눈에 띄게 만듭니다.

8 Packagist packages hid malicious package.json scripts, enabling Linux binary execution during installs and workflows.

"대신 PHP 코드와 함께 JavaScript 빌드 도구를 제공하는 프로젝트를 대상으로 package.json에 삽입되었습니다." 이러한 "생태계 간 배치"는 PHP 종속성을 검색하는 개발자와 보안 팀이 패키지 내에 번들로 제공되는 package.json 수명 주기 후크를 건너뛰고 Composer 관련 메타데이터에만 집중할 수 있기 때문에 활동을 눈에 띄게 만듭니다. 이후 악성 버전은 Packagist에서 제거되었습니다. 패키지를 분석한 결과 GitHub 릴리스 URL("github[.]com/parikhpreyash4/systemd-network-helper-aa5c751f")에서 Linux 바이너리를 다운로드하고 이를 "/tmp/.sshd" 폴더에 저장하고 "chmod"를 사용하여 권한을 변경하여 모든 사용자에게 실행 권한을 부여하고 백그라운드에서 실행하는 설치 후 스크립트를 포함하도록 업스트림 저장소가 수정된 것으로 나타났습니다. 패키지 이름과 관련 영향을 받는 버전은 아래에 나열되어 있습니다.
