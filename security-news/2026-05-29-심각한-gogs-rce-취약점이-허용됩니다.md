## Summary
Gogs 9.4 CVSS 결함은 1,141개의 노출된 인스턴스에서 git rebase 주입을 악용하여 원격 코드 실행을 가능하게 합니다. 인기 있는 오픈 소스 자체 호스팅 Git 서비스인 Gogs에서 인증된 사용자가 특정 조건에서 임의 코드를 실행할 수 있도록 허용하는 심각한 보안 취약점이 공개되었습니다. Rapid7에 따르면 이 보안 결함은 CVSS 점수 시스템에서 9.4점으로 평가되었습니다. CVE 식별자가 없습니다. 보안 연구원인 Jonah Burges는 "이 취약점을 통해 인증된 사용자는 '병합 전 리베이스' 병합 작업 중에 --exec 플래그를 git rebase에 삽입하는 악의적인 브랜치 이름이 포함된 풀 요청을 생성하여 서버에서 원격 코드 실행(RCE)을 달성할 수 있습니다."라고 말했습니다.

## Original Description

Gogs 9.4 CVSS flaw exploits git rebase injection on 1,141 exposed instances, enabling remote code execution.

## Excerpt
인기 있는 오픈 소스 자체 호스팅 Git 서비스인 Gogs에서 인증된 사용자가 특정 조건에서 임의 코드를 실행할 수 있도록 허용하는 심각한 보안 취약점이 공개되었습니다. Rapid7에 따르면 이 보안 결함은 CVSS 점수 시스템에서 9.4점으로 평가되었습니다. CVE 식별자가 없습니다. 보안 연구원인 Jonah Burgess는 "이 취약점을 통해 인증된 사용자는 '병합 전 리베이스' 병합 작업 중에 --exec 플래그를 git rebase에 삽입하는 악의적인 브랜치 이름이 포함된 풀 요청을 생성하여 서버에서 원격 코드 실행(RCE)을 달성할 수 있습니다."라고 밝혔습니다. 리베이스는 하나의 기능 브랜치에서 일련의 커밋을 가져와 다른 기본 브랜치 위에서 재생하여 선형 프로젝트 기록을 만드는 데 사용되는 Git 작업입니다. "git rebase"는 "git merge"와 동일한 문제(예: 한 브랜치의 변경 사항을 다른 브랜치에 통합)를 해결하는 반면, 전자는 원래 브랜치의 각 커밋에 대해 새 커밋을 생성하여 프로젝트 기록을 다시 작성합니다. "git rebase" 작업은 각 커밋이 재생된 후 실행되는 --exec 플래그를 통해 셸 명령을 인수로 받아들입니다. 이 취약점의 주목할만한 측면은 관리자 권한이나 다른 사용자와의 상호 작용이 필요하지 않다는 것입니다. 공격을 실행하기 위해 인증되지 않은 위협 행위자가 해야 할 일은 기본으로 구성된 인스턴스에 계정과 저장소를 생성하는 것뿐입니다.
