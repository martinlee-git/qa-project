## Summary
HTTP/2 Bomb은 HPACK 및 흐름 제어를 활용합니다. 단일 클라이언트는 20초 안에 32GB 메모리를 보유할 수 있으므로 서버 중단이 발생할 수 있습니다. 사이버 보안 연구원들이 NGINX, Apache HTTPD, Microsoft IIS, Envoy 및 Cloudflare Pingora를 포함한 주요 웹 서버에 영향을 미치는 원격 서비스 거부 공격을 발견했습니다. 이 취약점은 캘리포니아주에서 HTTP/2 Bomb로 코드명 붙여졌습니다. 회사는 “취약한 동작은 각 서버의 기본 HTTP/2 구성에 존재합니다.”라고 밝혔으며, OpenAI Codex는 이 취약점을 압축 폭탄과 Slowloris 스타일 홀드라는 두 가지 알려진 기술을 함께 연결하여 발견했다고 덧붙였습니다. "폭탄은 HTTP/2의 헤더 압축 체계인 HPACK을 표적으로 삼습니다. 즉, 전송된 1바이트가 1개의 전체 헤드가 됩니다…

## Original Description

HTTP/2 Bomb exploits HPACK and flow control; a single client can hold 32GB memory in 20 seconds, causing server outages.
## Key Points
- HTTP/2 Bomb은 HPACK 및 흐름 제어를 활용합니다. 단일 클라이언트는 20초 안에 32GB 메모리를 보유할 수 있으므로 서버 중단이 발생할 수 있습니다.
- 사이버 보안 연구원들이 NGINX, Apache HTTPD, Microsoft IIS, Envoy 및 Cloudflare Pingora를 포함한 주요 웹 서버에 영향을 미치는 원격 서비스 거부 공격을 발견했습니다.
- 이 취약점은 캘리포니아주에서 HTTP/2 Bomb라는 코드명으로 지정되었습니다.
## Excerpt
사이버 보안 연구원들이 NGINX, Apache HTTPD, Microsoft IIS, Envoy 및 Cloudflare Pingora를 포함한 주요 웹 서버에 영향을 미치는 원격 서비스 거부 공격을 발견했습니다. 이 취약점은 캘리포니아주에서 HTTP/2 Bomb로 코드명 붙여졌습니다. 회사는 “취약한 동작은 각 서버의 기본 HTTP/2 구성에 존재합니다.”라고 밝혔으며, OpenAI Codex는 이 취약점을 압축 폭탄과 Slowloris 스타일 홀드라는 두 가지 알려진 기술을 함께 연결하여 발견했다고 덧붙였습니다. "폭탄은 HTTP/2의 헤더 압축 방식인 HPACK을 표적으로 삼습니다. 즉, 전송되는 1바이트는 서버에서 하나의 전체 헤더 할당이 되며, 요청당 수천 번 반복됩니다."라고 Calif는 덧붙였습니다. "보류는 서버가 어떤 것도 해제하지 못하도록 하는 0바이트 흐름 제어 창입니다." HPACK은 Huffman 인코딩을 사용하여 요청 및 응답 메타데이터를 압축하는 데 사용되는 HTTP/2 전용 헤더 압축 알고리즘으로, 헤더 크기가 평균 30% 감소합니다. 또한 압축된 헤더에서 인증 쿠키를 유출할 수 있는 CRIME("Compression Ratio Info-leak Made Easy"의 약자)과 같은 공격에 탄력적으로 대응하도록 설계되었습니다.
