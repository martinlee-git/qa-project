# 공격자는 게시물에 Llm 에이전트를 사용합니다.

## Summary
LLM 기반 공격자는 2026년 5월 10일 CVE-2026-39987을 악용하여 자격 증명을 도용하고 PostgreSQL 데이터베이스를 유출했습니다. 최근 공개된 취약점을 이용해 공개적으로 접근 가능한 Marimo 네트워크를 악용한 후 초기 접근 권한을 획득한 후, 알려지지 않은 위협 행위자가 LLM(대형 언어 모델) 에이전트를 사용하여 침해 후 조치를 수행하는 것이 관찰되었습니다. "공격자는 CVE-2026-39987을 통해 인터넷에 연결할 수 있는 Marimo 노트북을 손상시키고 손상된 호스트에서 2개의 클라우드 자격 증명을 추출하고 팬아웃 송신 풀을 통해 이를 재생하여 AWS Secrets Manager에서 SSH 개인 키를 검색하고 해당 키를 사용하여 다운스트림에 대해 8개의 짧은 SSH 세션을 구동했습니다.

## Original Description

LLM-driven attackers exploited CVE-2026-39987 on May 10, 2026, to steal credentials and exfiltrate a PostgreSQL database.

## Excerpt
최근 공개된 취약점을 이용해 공개적으로 접근 가능한 Marimo 네트워크를 악용한 후 초기 접근 권한을 획득한 후, 알려지지 않은 위협 행위자가 LLM(대형 언어 모델) 에이전트를 사용하여 침해 후 조치를 수행하는 것이 관찰되었습니다. Sysdig는 "공격자는 CVE-2026-39987을 통해 인터넷에 연결할 수 있는 Marimo 노트북을 손상시키고 손상된 호스트에서 2개의 클라우드 자격 증명을 추출하고 이를 팬 아웃된 송신 풀을 통해 재생하여 AWS Secrets Manager에서 SSH 개인 키를 검색한 다음 해당 키를 사용하여 다운스트림 SSH 배스천 서버에 대해 8개의 짧은 SSH 세션을 구동했습니다"라고 말했습니다. "배스천 단계에서는 2분 이내에 내부 PostgreSQL 데이터베이스의 스키마와 전체 콘텐츠를 유출했습니다." CVE-2026-39987은 0.20.4 이전의 모든 Marimo 버전에 영향을 미치는 중요한 사전 인증 원격 코드 실행 취약점을 나타냅니다. 이를 통해 인증되지 않은 공격자가 임의의 시스템 명령을 실행할 수 있습니다. 이 문제는 지난달 출시된 버전 0.23.0에서 해결되었습니다. 이후 이 보안 결함은 공격자들이 이를 이용해 허니팟 시스템에 대한 수동 정찰을 시작하고 민감한 데이터를 수집하는 등 적극적으로 악용되었습니다.
