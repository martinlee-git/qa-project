## Summary
4가지 DifyTap 결함으로 인해 Dify 테넌트 전체의 비공개 AI 채팅 및 파일이 노출될 수 있습니다. 3개는 버전 1.14.2에서 수정되었습니다. 사이버 보안 연구원들은 146,000개 이상의 GitHub 스타가 포함된 오픈 소스 에이전트 워크플로 플랫폼인 Dify의 4가지 취약점에 대한 세부 정보를 공개했습니다. 이 취약점을 통해 공격자는 인증 없이 다른 고객의 애플리케이션에서 인공 지능(AI) 변환을 은밀하게 읽을 수 있습니다. 이 취약점은 Zafran Security에 의해 총체적으로 DifyTap이라는 코드명으로 지정되었습니다. "2개는 심각한 심각도였고, 2개는 인증이 필요하지 않았으며, 3개는 Dify의 다중 테넌트 클라우드 서비스에 테넌트 간 영향을 주어 한 고객의 데이터가 다른 서버에 노출될 수 있었습니다.

## Original Description

Four DifyTap flaws could expose private AI chats and files across Dify tenants; three are fixed in version 1.14.2.

## Excerpt
사이버 보안 연구원들은 146,000개 이상의 GitHub 스타가 포함된 오픈 소스 에이전트 워크플로 플랫폼인 Dify의 4가지 취약점에 대한 세부 정보를 공개했습니다. 이 취약점을 통해 공격자는 인증 없이 다른 고객의 애플리케이션에서 인공 지능(AI) 변환을 은밀하게 읽을 수 있습니다. 이 취약점은 Zafran Security에 의해 총체적으로 DifyTap이라는 코드명으로 지정되었습니다. 연구원 Ido Shani와 Gal Zaban은 "2개는 심각도가 심각했고, 2개는 인증이 필요하지 않았으며, 3개는 Dify의 다중 테넌트 클라우드 서비스에 테넌트 간 영향을 미쳐 한 고객의 데이터가 다른 고객에게 노출될 수 있었습니다"라고 말했습니다. 보안 결함으로 인해 공격자는 다른 고객의 애플리케이션에서 비공개 AI 채팅을 읽고 모든 메시지 및 모델 응답에 대한 은밀한 유출 채널을 만들 수 있었습니다. 또한 인증되지 않은 요청에서 Dify의 내부 플러그인 데몬 API를 통과하고 테넌트 간 내부 API 호출을 트리거할 수 있을 뿐만 아니라 다른 테넌트가 업로드한 문서를 미리 보고 다른 사용자의 파일 고유 식별자를 연결하여 테넌트 내 사용자 간에 파일을 유출할 수 있게 되었습니다.
