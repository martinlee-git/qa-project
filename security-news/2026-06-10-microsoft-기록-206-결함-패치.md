## Summary
Microsoft는 39개의 심각한 결함과 3개의 공개된 제로데이를 포함하여 소프트웨어 포트폴리오 전반에 걸쳐 206개의 취약점에 대한 수정 사항을 발표했습니다. Microsoft는 화요일에 출시 당시 공개된 세 가지 결함을 포함하여 소프트웨어 포트폴리오에 영향을 미치는 기록적인 206개의 보안 취약점에 대한 수정 사항을 발표했습니다. 206개 결함 중 39개는 심각, 167개는 심각도가 중요로 평가되었습니다. 여기에는 권한 상승 63건, 원격 코드 실행 56건, 정보 공개 30건, 스푸핑 27건, 보안 기능 우회 20건, 서비스 거부 7건, 변조 취약점 3건이 포함됩니다. 패치에는 Microsoft가 아닌 타사 CVE 2개, 즉 권한 상승 취약점도 포함되어 있습니다.

## Excerpt
Microsoft는 화요일에 출시 당시 공개된 세 가지 결함을 포함하여 소프트웨어 포트폴리오에 영향을 미치는 기록적인 206개의 보안 취약점에 대한 수정 사항을 발표했습니다. 206개 결함 중 39개는 심각, 167개는 심각도가 중요로 평가되었습니다. 여기에는 권한 상승 63건, 원격 코드 실행 56건, 정보 공개 30건, 스푸핑 27건, 보안 기능 우회 20건, 서비스 거부 7건, 변조 취약점 3건이 포함됩니다. 또한 패치에는 두 가지 타사 CVE, Windows 커널에 영향을 미치는 권한 상승 취약점(CVE-2025-10263) 및 UEFI 보안 부팅 보안 기능 우회(CVE-2026-8863)가 포함되어 있습니다. 이는 Google이 Microsoft Edge 브라우저에 사용되는 Chromium에서 해결한 350개 이상의 보안 결함에 추가됩니다. 수정 사항 목록의 1위는 CVE-2026-45657(CVSS 점수: 9.8)로, Windows 커널에 영향을 미치고 원격 코드 실행을 초래할 수 있는 use-after-free 결함입니다. 마이크로소프트는 “공격자는 특수 제작된 네트워크 트래픽을 취약한 윈도우 시스템에 전송함으로써 이 취약점을 악용할 수 있다”고 밝혔다. "성공할 경우 악성 네트워크 패킷은 Windows 커널이 특정 TCP/IP 데이터를 처리하는 방식에 결함을 유발할 수 있으며, 잠재적으로 공격자가 로그인하거나 사용자와 상호 작용하지 않고도 시스템 수준 권한으로 코드를 실행할 수 있습니다."
