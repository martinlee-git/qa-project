## Summary
ServiceNow 보안 문제로 인해 특정 상황에서 인증되지 않은 사용자가 취약한 인스턴스에 의도한 것보다 더 많은 액세스 권한을 얻을 수 있었습니다. ServiceNow는 알려지지 않은 위협 행위자가 취약한 인스턴스에 대한 더 깊은 무단 액세스를 얻기 위해 결함을 악용한 보안 사고에 대해 경고했습니다. 회사는 고객 액세스가 필요한 권고를 통해 "2026년 6월 5일에 호스팅된 고객 인스턴스에 보안 업데이트를 적용했습니다."라고 밝혔습니다. "이 업데이트는 특정 상황에서 인증되지 않은 사용자가 의도한 것보다 ServiceNow 인스턴스에 더 많은 액세스 권한을 얻을 수 있는 보안 문제와 관련이 있습니다." 보안 업데이트는 끝점 구성을 제한적으로 변경합니다.

## Original Description

A ServiceNow security issue allowed unauthenticated users, in certain circumstances, to gain greater access to susceptible instances than intended.
## Key Points

- ServiceNow는 알려지지 않은 위협 행위자가 취약한 인스턴스에 대한 더 깊은 무단 액세스를 얻기 위해 결함을 악용한 보안 사고에 대해 경고했습니다.
- 회사는 고객 액세스가 필요한 권고를 통해 "2026년 6월 5일에 호스팅된 고객 인스턴스에 보안 업데이트를 적용했습니다."라고 밝혔습니다.
## Excerpt
ServiceNow는 알려지지 않은 위협 행위자가 취약한 인스턴스에 대한 더 깊은 무단 액세스를 얻기 위해 결함을 악용한 보안 사고에 대해 경고했습니다. 회사는 고객 액세스가 필요한 권고를 통해 "2026년 6월 5일에 호스팅된 고객 인스턴스에 보안 업데이트를 적용했습니다."라고 밝혔습니다. "이 업데이트는 특정 상황에서 인증되지 않은 사용자가 의도한 것보다 ServiceNow 인스턴스에 더 많은 액세스 권한을 얻을 수 있는 보안 문제와 관련이 있습니다." 보안 업데이트는 엔드포인트 구성을 변경하여 이 액세스를 인증된 사용자로 제한합니다. 보안 결함에는 현재 CVE 식별자가 없습니다. 이 문제에 대한 자세한 내용은 Reddit에서 처음 나타났습니다. ServiceNow는 보안 문제와 관련된 비정상적인 활동을 감지했으며 "일부 고객"에 대한 인스턴스 테이블 쿼리에 성공한 증거를 관찰했다고 밝혔습니다. 영향을 받은 고객에게 통보했다고 덧붙였습니다. "보안 문제는 호주 플랫폼 릴리스를 사용하거나 호주 이전 릴리스의 인스턴스에 특정 구성을 변경한 고객과 관련이 있습니다."라고 지적했습니다.
